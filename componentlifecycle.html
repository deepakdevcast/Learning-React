Component Lifecycle

Mainly Component have 4 level in Lifcycle

1. Mounting - When instance of Component created and inserted in DOM
methods in mounting: constructor, static getDerivedStateFromProps, render and componentDidMount

2. Updating - When Component re-rendering on either change its props or state
methods in updating: static getDerivedStateFromProps, shouldComponentUpdate, render, getSnapshotBeforeUpdate and componentDidUpdate

3. Unmounting - when Component removed from DOM methods in unmounting: componentWillUnmount

4. Error Handling - When their is error on rendering component,in the lifecycle method or in the constructor of any child component methods in error handling: static getDerivedStateFromError and componentDidCatch

1. Mounting

execute order : constructor -> getDerivedStateFromProps -> render() -> componentDidMount

constructor() :

point- i. special function that will get called whenever new component created
ii. initializing state
iii. binding the event handlers
iv. do not cause side effects. ex: HTTP request
v. super(props) and directly overwrite this.state

getDerivedStateFromProps(props,state):

point-i. when the state of the component depends on changed in props over time.
ii. set the state
iii. do not cause side effects. ex: HTTP request

render():

point- i. Only required method
ii. read props & state and return JSX
iii. do not change state or interact with DOM or make ajax calls
iv. Children component lifecycle methods are also executed

componentDidMount()

point- i. Invoke immediately after a component and all its children components have been rendered to the DOM.
ii. you can cause side effects in this. ex: Iterect eith the DOM or perform any ajac calls to load data.

2. Updating

execute order : getDerivedStateFromProps->shouldComponentUpdate->render->getSnapshotBeforeUpdate -> componentDidUpdate

static getDerivedStateFromProps(props,state) :

point- i. Method is called every time a component is re-rendered
ii. set the state
iii. do not cause side effects. ex: HTTP request

shouldComponentUpdate(nextProps,nextState):

point-i. Dictated if the component should re-render or not
ii. used for performance optimization
iii. do not cause side effects. ex: HTTP requests, calling the setState method

render():

point- i. Only required method
ii. read props & state and return JSX
iii. do not change state or interact with DOM or make ajax calls
iv. Children component lifecycle methods are also executed

getSnapshotBeforeUpdate(preProps,precState):

point- i. Called right before the change from the virtual DOM are to be reflected in the DOM.
ii. used to capture some information from the DOM ex: maintained the scroll position after the update
iii. Method will either return null or return a value. Returned value will be passed as the parameter to the next method.

componentDidUpdate(preProps,precState,snapshot):

point- i. Called after the render is finished in the re-render cycles
ii. You can cause side effects ex: ajax calling

3. Unmounting

componentWillUnmount():

point-i. Methed is invoked immediately before a component is unmounted and destroyed.
ii. Can be used in- cancelling any network requests, removing event handlers, cancelling any subscriptions and also invalidating timers.
iii. Do not call setState method.

4. Error Handling

static getDerivedStateFromError(error),
componentDidCatch(error,info)
both method call when ther is an error either during rendering, in a lifecycle method, or in the constructor of any child component.
